#  Star Wars Async Data Loader

Асинхронный скрипт для выгрузки персонажей из Star Wars API и сохранения в SQLite. 
Проект реализован с учётом требований к устойчивости, разделению миграций и полной асинхронности.

## ✨ Особенности
- 🔄 **Пагинация API**: обход всех страниц через поле `next`, а не фиксированный диапазон ID
- ⚡ **Полная асинхронность**: HTTP-запросы и запись в БД не блокируют event loop
- 🛡️ **Устойчивость к сбоям**: автоматические повторные попытки при таймаутах и ошибках сети
- 📦 **Разделение ответственности**: миграция схемы БД вынесена в отдельный скрипт
- 🌍 **Трансформация данных**: URL планеты заменяется на её название (`homeworld`)
- ✅ **Контроль целостности**: проверка на дубликаты и заполненность полей перед коммитом

## 🛠 Технологии
- Python 3.10+
- `aiohttp` — асинхронные HTTP-запросы
- `SQLAlchemy` + `aiosqlite` — асинхронная ORM и драйвер SQLite
- `asyncio` — управление конкурентными задачами

## 📋 Установка

pip install -r requirements.txt

## 🚀 Запуск

1. Создание схемы БД (безопасно при повторном запуске):

python migrate.py

2. Загрузка данных из API:

python load_data.py

## 📊 Результат

В базе starwars.db создаётся таблица characters

Загружено 82 персонажа (все доступные в API)

Поле homeworld содержит название планеты, а не URL

Все обязательные поля заполнены: id, birth_year, eye_color, gender, hair_color, homeworld, mass, name, skin_color

## 📝 Примечания

API: https://www.swapi.tech/api/people/

Миграция использует create_all, что гарантирует безопасный повторный запуск без ошибок

Сетевые запросы защищены механизмом retry (до 3 попыток с экспоненциальной задержкой)

Загрузка происходит пачками по 3–5 персонажей с паузами, чтобы не перегружать API

Файл starwars.db создаётся автоматически при первом запуске load_data_async.py
